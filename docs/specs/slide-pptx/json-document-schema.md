# JsonDocument スキーマ仕様書

このドキュメントは、`json_document` アーティファクトタイプのJSONデータの仕様を定めたものです。AI（LLM）がビジネス文書や構造化ドキュメントを生成・エクスポート（Word出力等）する際のガイドラインとして機能します。

## 1. 基本構造 (Root Object)

ドキュメントのデータ全体は、以下のルートオブジェクトで構成されます。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `meta` | object | 任意 | 文書のメタデータ（ヘッダー情報、宛名、件名など） |
| `blocks` | array | **必須** | ドキュメントの本文を構成するブロック要素の配列 |

---

## 2. メタデータ (`meta`)

文書の全体的な属性や、ビジネスレターのヘッダー情報を定義します。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `template` | string | 任意 | `"letter"`（ビジネスレター形式）または指定なし。`"letter"`を指定すると、ビジネス向けのフォーマルな白黒基調スタイルが適用されます。 |
| `title` | string | 任意 | 文書のメインタイトル・件名 |
| `date` | string | 任意 | 文書の日付テキスト（例: "2026年3月26日"） |
| `refNo` | string | 任意 | 文書番号・リファレンス番号（例: "TM-2026-03-01"） |
| `recipient` | object \| string | 任意 | 宛名情報。文字列、または `{postal, address, company, name}` を持つオブジェクト。 |
| `sender` | object \| string | 任意 | 差出人情報。文字列、または `{postal, address, company, department}` を持つオブジェクト。 |

---

## 3. ブロック要素 (`blocks`)

ドキュメントの本文を構成する各要素（段落、見出し、リスト、テーブル等）です。`blocks` 配列内に順序通りに定義します。

### 3.1. 見出し (`heading`)
文書のセクションタイトル。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"heading"` を指定 |
| `level` | number | **必須** | 見出しのレベル（`1`, `2`, `3` などを推奨） |
| `text` | string | **必須** | 見出しのテキスト |

### 3.2. リッチテキスト・通常段落 (`rich_text`)
テキスト本文や、装飾ボックスなどの表現に用います。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"rich_text"` を指定 |
| `text` | string | **必須** | 本文のテキスト。マークダウンの太字（`**テキスト**` または `<strong>テキスト</strong>`）による部分的な太字化をサポートしています。 |
| `variant` | string | 任意 | 装飾ボックスの指定（`"notice-box"`, `"notice-side"`, `"warning"`, `"success"`, `"info"` など）。指定しない場合は通常の段落としてレンダリングされます。 |
| `title` | string | 任意 | `variant` を指定した装飾ボックス内に表示されるタイトル（例: `"重要事項"`）。この値が設定されていると、上部中央に太字でタイトルが描画され、その下に区切り線が自動追加されます。 |

### 3.3. 箇条書きリスト (`list`)
箇条書き（箇条書きマーク付き）のリスト。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"list"` を指定 |
| `items` | array | **必須** | リスト項目となる文字列の配列 |

### 3.4. テーブル (`table`)
表形式のデータ表示。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"table"` を指定 |
| `headers` | array | 任意 | 列見出し（ヘッダー行）となる文字列の配列 |
| `rows` | array | **必須** | 行データの配列。各要素は文字列の配列（セルデータ）です。 |

### 3.5. グラフ (`chart`)
Rechartsライブラリを使用した美しいグラフの描画。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"chart"` を指定 |
| `chart_type` | string | 任意 | `"bar"`, `"line"`, `"pie"` のいずれか（デフォルト: `"bar"`） |
| `title` | string | 任意 | グラフの上部に表示されるタイトル |
| `height` | number | 任意 | グラフの高さ（ピクセル単位、デフォルト: `300`） |
| `data` | array | **必須** | データポイントの配列。各要素は `{name: string, value: number}` の形式を持つオブジェクト。 |

### 3.6. SVG画像 (`svg`)
Mermaidなどで生成したSVG形式のダイアグラムや図をそのまま表示します。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"svg"` を指定 |
| `content` | string | **必須** | SVG要素の生テキスト（`<svg>...</svg>`） |

### 3.7. ページ区切り (`page_break`)
明示的な改ページを挿入します（Word出力時にページが切り替わります）。

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `type` | string | **必須** | `"page_break"` を指定 |

---

## 4. 特記事項

- **太字のサポート**: `rich_text` の `text` 内では、部分的なテキストの太字（`**強調**` など）がサポートされており、Webプレビューおよび `.docx` 出力の両方でシームレスに再現されます。
- **ビジネスレターの最適化**: `meta.template` に `"letter"` を指定すると、各種ボックスの枠線や見出しがフォーマルな黒・グレー基調へ自動調整され、インク資源に配慮したデザインでWord出力されます。
