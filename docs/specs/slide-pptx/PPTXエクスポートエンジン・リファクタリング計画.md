# **PPTXエクスポートエンジン リファクタリング計画**

## **1\. 背景と現状の課題**

現在の PptxExportEngine は、React側でレンダリングされるJSONスライド（Web表示）の構造をPowerPointファイル（PPTX）として出力する役割を担っています。しかし、WebブラウザのレイアウトエンジンとPPTXの静的描画仕様の根本的な違いにより、一部のレイアウトバリエーションで表示の差異や破綻が生じているのが現状です。

### **1.1 パラダイムのギャップ**

* **React (Web) のレイアウト:** Flexbox、Grid、CSS Column、transform: scale 等を用いた**動的・相対的**な自動計算。画面幅やテキスト量に柔軟に適応する。  
* **PptxGenJS (PPTX) のレイアウト:** 全要素に対する x, y, w, h（座標とサイズ）の**静的・絶対的**な指定。

### **1.2 具体的な課題例**

* **スタイルのハードコードと二重管理:** PPTX側のコード内にカラーコード（6366F1など）が直接記述されており、React側のテーマ変更に追従しにくい。  
* **レイアウト計算の再発明:** インサイトボックスやカードの描画ロジックが、各スライドメソッド (renderChartSlide, renderStatsSlide など) で個別に実装されている。  
* **レイアウトバリエーションへの対応漏れ:**  
  * ChartSlide: 左右分割（two-column）時にインサイトボックスの枠線有無をif文で強引に処理しており、React側のCSS変更に弱い。  
  * ContentSlide: React側にある2段組み（columnCount: 2）の概念がPPTX側に実装されておらず、常に1カラムで出力される。  
  * SplitSlide: React側の自動縮小（情報量に応じた transform: scale）が考慮されておらず、文字量が多いとPPTX上で文字が枠から溢れてしまう。

## **2\. 解決へのアプローチ：実装計画**

これらの課題を根本から解決するため、「PPTX描画ロジックの抽象化」と「レイアウト計算の共通化」を目的とした4段階のリファクタリングを実施します。

### **フェーズ 1：テーマとスタイルの統合管理**

ハードコードされているスタイル情報を抽出し、テーマオブジェクトとして一元管理します。

* **目的:** デザイン変更時の影響範囲を局所化し、React側のテーマ定義との同期を容易にする。  
* **実装内容:**  
  * PptxExportEngine 内にテーマ定数（Primary、Secondary、Text、Borderの色や、標準のフォントサイズなど）を定義。  
  * parseRichText に渡すベーススタイル定義の共通化。

### **フェーズ 2：UIコンポーネント（ビルディングブロック）の共通化**

複数のスライドで繰り返し登場するUI要素を、独立した描画メソッドとして切り出します。

* **目的:** コードの重複を排除し、デザインの一貫性を担保する。  
* **実装内容:** 以下の共通描画メソッドを作成  
  * renderSlideHeader: タイトルとアンダーライン  
  * renderInsightBox: 背景色・左アクセント線付きのハイライトボックス  
  * renderCard: 角丸・ボーダー付きの背景矩形  
  * renderFooterAnnotations: スライド下部の注釈テキスト

### **フェーズ 3：レイアウト計算エンジンの導入**

Webの柔軟なレイアウトをPPTX上でシミュレートするためのユーティリティ関数群を実装します。**（本リファクタリングのコア部分）**

* **目的:** 絶対座標の計算を抽象化し、複雑なレイアウト（多段組みやスケーリング）を安全に実現する。  
* **実装内容:**  
  * **Grid / Column System:** 全幅・ギャップ・カラム数を元に、各カラムの x, w 座標配列を返す関数。  
  * **Vertical Alignment:** 描画領域とコンテンツの総高さを元に、垂直中央寄せの開始Y座標（startY）を算出する関数。  
  * **Auto Scaling System:** テキスト量（行数・文字数）から情報密度を計算し、全体に適用するスケール係数（擬似的な transform: scale）を算出する関数。

### **フェーズ 4：各スライド描画メソッドのリファクタリング**

構築したビルディングブロックとレイアウトエンジンを用いて、既存のスライド描画メソッドを全面的に書き換えます。

* **目的:** すべてのレイアウトバリエーションに網羅的に対応し、Reactの表示に限りなく近づける。  
* **実装内容:**  
  * renderChartSlide: isTwoColumn フラグに応じてレイアウトエンジンで領域を分割し、renderInsightBox を適用。  
  * renderContentSlide: Column Systemを用いて、1カラム・2カラムのテキスト流し込みを動的に処理。  
  * renderSplitSlide: Auto Scaling Systemを適用し、文字溢れを防止。

## **3\. 今後の進め方**

まずは基盤となるフェーズ1（テーマ統合）**および**フェーズ2（ビルディングブロックの共通化）から着手します。これにより、コードの可読性が飛躍的に向上し、フェーズ3以降の複雑なレイアウト計算を導入しやすい状態を作ります。
