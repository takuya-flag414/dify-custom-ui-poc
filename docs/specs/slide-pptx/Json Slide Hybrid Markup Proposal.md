# **Json Slide: Markdown \+ インラインHTML ハイブリッド実装案**

## **1\. 概要と目的**

現状の json\_slide タイプのテキストフィールドはプレーンテキストベースであり、太字、斜体、文字色、リスト表示などのリッチな表現が制限されています。また、配列構造（bullet\_points など）を持つため、AIの生成トークン数が増加し、JSONスキーマの複雑さがエラーを引き起こすリスクがあります。

本提案は、AI（LLM）が得意とする **Markdown** と、表現力の高い **インラインHTML** を組み合わせたハイブリッドなマークアップルールを導入するアーキテクチャ設計です。

**期待される効果**:

1. スライド表現のリッチ化（テキスト内の一部の文字色変更やハイライト）  
2. JSONスキーマのフラット化（トークン節約・エラー率低下）  
3. 将来的なWYSIWYGエディタ体験の向上  
4. **デザインシステムによる統一感の担保**（事前定義されたカラーパレットへの制限）

## **2\. ハイブリッド案の機能概要**

1. **AIの出力負荷を最小化**: 基本的な装飾（改行、強調、箇条書き）はAIが得意なMarkdownを使用し、ネストした配列構造（bullet\_pointsなど）を単一のテキストフィールド（body\_text 等）に統合します。  
2. **デザインガイドラインの強制**: style="..." による直接の色指定を禁止し、**アプリ側で事前定義されたユーティリティクラスのみ**を許可します。これにより、AIが自由に装飾してもスライドの統一感（ブランドカラー）が損なわれません。  
3. **セキュアなレンダリング**: フロントエンド側でマークダウンをパースする際、サニタイズ処理を挟み、XSSリスクを排除しつつ許可されたタグとクラスのみを適用します。

## **3\. マークアップ仕様（ルール定義）**

AIに対して以下のルールをプロンプトで厳格に指示し、スライドの各テキストフィールドに出力させます。

### **3.1 許可されるMarkdown構文**

* **改行**: \\n または \\n\\n  
* **太字**: \*\*太字\*\*  
* **斜体**: \*斜体\*  
* **箇条書きリスト**: 行頭に \- または \*  
* **番号付きリスト**: 行頭に 1\.

### **3.2 許可されるインラインHTMLと事前定義カラーパレット**

文字色や背景色のハイライトには、\<span\> や \<mark\> タグに特定のクラスを付与することのみを許可します。**style 属性によるHexコード（\#FFFFFFなど）やRGBの直接指定は厳禁**とします。

**【許可されるHTMLタグ】**

* \<span\> (文字色変更用)  
* \<mark\> (背景ハイライト用)  
* \<br\> (強制改行用)  
* \<u\> (下線用)

**【事前定義カラーパレット（ユーティリティクラス）】**

これらのクラスはスライドの「テーマ」に連動して動的に色が切り替わります。

* **文字色**  
  * text-primary: プライマリカラー（ブランドのメインカラー）  
  * text-accent: アクセントカラー（注意を引く色）  
  * text-muted: ミュートカラー（補足情報用の薄い色）  
  * text-success: 成功・ポジティブな色（緑系）  
  * text-danger: 警告・ネガティブな色（赤系）  
* **背景ハイライト**  
  * bg-highlight: プライマリカラーの薄い背景色  
  * bg-warning: 注意を促す薄い背景色

**記述例（AIの出力イメージ）**:

今期の最重要課題は**顧客体験の向上**です。\\n特にオンラインサポートのレスポンス改善が急務となっています。\\n\\n- サポート時間の短縮\\n- 顧客満足度の向上

## **4\. データ構造のスリム化（スキーマ変更）**

既存の配列構造をMarkdownテキストに統合し、スキーマをフラットにします。

### **Before (従来の構造)**

{  
  "layout\_type": "content\_slide",  
  "content": {  
    "title": "課題と解決策",  
    "key\_message": "コストの高止まりを解消します",  
    "body\_text": "現状の課題はコストの高止まりです。",  
    "bullet\_points": \[  
      "自動化ツールの導入",  
      "業務プロセスの見直し"  
    \]  
  }  
}

### **After (ハイブリッド・マークアップ適用後)**

{  
  "layout\_type": "content\_slide",  
  "content": {  
    "title": "課題と解決策",  
    "body\_text": "\*\*コストの高止まり\*\*を解消します。\\n現状の課題はコストの高止まりです。\\n\\n- 自動化ツールの導入\\n- \<span class='text-primary'\>業務プロセスの見直し\</span\>"  
  }  
}

※ key\_message や bullet\_points などの細分化されたフィールドを削除し、大まかなブロック（body\_text, left\_column, right\_column など）にMarkdownとして集約します。

## **5\. フロントエンドでのレンダリング実装方針（Viewer側）**

React等のフロントエンドでこのデータを受け取った際の描画方針です。

1. **Markdownパーサーの利用**: react-markdown などのライブラリを使用し、JSON内のテキストをHTMLに変換してレンダリングします。  
2. **HTMLパースプラグイン**: rehype-raw 等を組み合わせ、AIが出力した \<span\> や \<mark\> などのインラインHTMLを安全に解釈させます。  
3. **サニタイズ処理 (重要)**:  
   * 許可リスト（Allowed Tags / Allowed Classes）以外のタグ（\<script\>, \<iframe\>など）を除外します。  
   * **style 属性を完全にサニタイズ（削除）** し、事前定義された class のみを残すことで、デザインの崩れや意図しない色の注入を防ぎます。  
4. **テーマCSSとの連携**: スライドのテーマCSS（corporate-modernなど）内に、.text-primary や .bg-highlight といったユーティリティクラスを定義し、現在選択されているテーマの変数（var(--slide-primary)など）を参照させます。

## **6\. SlideFormEditor 側の実装方針（エディタ側）**

現状の \<textarea\> を、マークダウンおよび事前定義スタイル対応のエディタに置き換えます。

### **ステップ1: プレビュー付きMarkdownエディタ（短期）**

入力をプレーンな文字列として扱い、Markdown構文とHTMLタグを直接編集させます。すぐ横にレンダリング結果のプレビューを表示することで、ユーザーが確認しながら微調整できるようにします。

### **ステップ2: カスタムWYSIWYGエディタの導入（中・長期）**

TipTap 等のヘッドレスエディタを組み込みます。

* 文字色変更ボタンなどをUI上に用意しますが、カラーパレットから任意の色を選ばせるのではなく、「プライマリ」「アクセント」といった**事前定義された意味的なカラー（Semantic Colors）の選択肢のみ**を提供します。  
* エディタの出力形式としてMarkdown \+ 許可されたHTMLクラスを出力するように設定し、JSONデータとの互換性を保ちます。
