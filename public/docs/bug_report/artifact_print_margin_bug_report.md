# ArtifactUI 印刷プレビュー「右寄り（軸ずれ）」バグ報告書

本報告書は、AI（LLM）から返却されたHTMLアーティファクトにおいて「パネル内の画面プレビューでは問題ないのに、印刷プレビューにするとドキュメント全体が右に偏って見切れてしまう」現象についての分析結果です。

---

## 現象と原因の結論

### 症状
ブラウザ固有の「印刷プレビューダイアログ」を開いた段階で、A4用紙（210mm幅）に対してテキスト等の中身すべてが右方向にズレており、左側に不自然な余白ができている。

### 原因
LLMが出力したHTML・CSSの中に、**「印刷時のブラウザのデフォルト余白を無効化する記述」が抜け落ちていた** ことが原因です。
同時に「要素の横幅を210mmに強制する」記述は存在していたため、両者が衝突して物理的に右へはみ出す結果となっていました。

詳細なメカニズムは以下の通りです：

1. **ブラウザ自体のデフォルト余白の存在**
   Chromeなどのブラウザで印刷しようとした際、最初から用紙の上下左右に「約10mm程度の標準余白」が自動的に設定されます。
   本来、これを無効化するためには以下のように `@page` 規則で `margin: 0;` を宣言する必要があります。
   ```css
   /* ガイドラインには記載されているが、LLMが出力し忘れたコード */
   @page { size: A4; margin: 0; }
   ```

2. **LLMは「横幅210mm」だけをハードコードした**
   LLMは `@page` の余白リセット指示を書き忘れた（あるいはトークン節約等の理由で削った）にもかかわらず、印刷時用の専用スタイルとして以下のように「横幅ぴったり210mm」を指定していました。
   ```css
   /* LLMが実際に出力したコード（抜粋） */
   @media print {
     .page { 
       width: 210mm; /* A4の横幅と同じサイズ */
     }
   }
   ```

3. **「ブラウザ余白」＋「幅210mmの要素」が合体して右へズレる**
   ブラウザは「左端から10mmの標準余白」を空けた場所をスタート地点として、LLMから指示された「幅210mmの `.page` ボックス」を配置しようとします。
   結果として、ボックスの右端の位置は `10mm（余白） + 210mm（中身） = 220mm` となり、実際のA4用紙の幅（210mm）を突破してしまいます。これが「ドキュメントの軸が中心からズレて、右に偏り、右側が見切れる」というバグの実態です。

### 解決策（推奨される対応）
前のバグ報告書でも挙げたとおり、このガイドライン（`html-artifact-guide.md`）自体に「LLMにピクセルやミリメートル単位での絶対値でのスタイリングを強制させすぎている」という問題があります。

* `.page` に対して `width: 210mm;` や `height: 296.8mm;` といった物理的な絶対サイズ（mm指定）による縛りを廃止する。
* もしくは、LLMの出力に依存せず、フロントエンドの React/CSS 側でプレビュー用のコンテナ調整や印刷用のスタイルリセット（`@page { margin: 0; }` など）を包括的に適用・上書きして保護するようにする。
