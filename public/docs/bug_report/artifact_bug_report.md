# ArtifactUI プレビューバグ報告書

本報告書は、AI（LLM）から返却されたHTMLアーティファクトをレンダリングする `ArtifactPanel.jsx` で発生した2件の表示不具合についての原因と解決策をまとめたものです。

---

## 1. プレビューの「白い用紙部分」が縦に伸び続けるバグ

### 症状
HTML型アーティファクトのプレビュー時、白い用紙部分がスクロール領域として下方向に無限に伸びてしまう（画面がストレッチし続ける）現象が発生。

### 原因
フロントエンド側（`ArtifactPanel.jsx`）の `iframe` の高さ自動調整ロジックにおける**無限リサイズループ**が原因。
LLMが生成したHTMLコンテンツ側で `min-height: 100vh;` 等が与えられており、親要素（iframe）の高さに自身を合わせようとします。一方でフロントエンド側は、受信したドキュメントサイズ (`scrollHeight`) に `+ 12` ピクセルの固定余白を足してiframeの高さを更新していました。
```javascript
// 原因となったコード（抜粋）
const newHeight = Math.max(e.data.height + 12, 297 * 3.7795);
```
これにより、「iframeが+12px広がる」→「内部のコンテンツがそれに合わせて+12px広がる」→「ResizeObserverが+12pxされた高さを親に通知する」→「親がさらに+12pxする」というフィードバックループが発生していました。

### 解決策（対応完了）
`ArtifactPanel.jsx` にて、高さを設定する際の `+ 12` ピクセルの加算処理を削除しました。
```diff
- const newHeight = Math.max(e.data.height + 12, 297 * 3.7795);
+ const newHeight = Math.max(e.data.height, 297 * 3.7795);
```
これにより、コンテンツ自身の高さを超えて領域が膨張することがなくなりました。

---

## 2. プレビューの上下に「グレーの余白」が挿入されるバグ

### 症状
生成されたアーティファクトのプレビュー時、本来の「A4用紙（白）」の外側（上下）にグレーの不要な余白が表示されてしまう。

### 原因
**LLM側（プロンプト起因）の出力するHTML内のCSS** が原因。
LLMが「A4ドキュメントのプレビュー画面らしさ」を考慮しすぎてしまい、生成するHTML自体の `body` タグに対して背景色や上下のPadding（余白）を自己完結で直接指定してしまっていました。
```css
/* LLMが生成してしまったCSS */
html, body {
  background: #f0f2f5; /* グレーの背景色 */
}
body {
  padding: 15mm 0; /* 上下に15mmの余白 */
}
```
フロントエンド側はHTMLの `scrollHeight` に応じて `iframe` を広げるため、このLLMが意図的に作成したグレーの余白領域がそのまま表示されていました。

### 解決策（今後の対応）
表示側のフロントエンドUIではなく、アーティファクトを生成する**LLM側のシステムプロンプトや出力フォーマット指示**を修正・徹底する必要があります。
以下の制約をAIのプロンプトに追加することを推奨します：
* 「`html` や `body` タグには `margin` や `padding` を設定しないこと。」
* 「背景は必ず背景色なし（または白）に固定すること。」
* 「デザイン上の外枠や余白は、生成するHTML内では表現しないこと（フロントエンド側でラップするため）。」
