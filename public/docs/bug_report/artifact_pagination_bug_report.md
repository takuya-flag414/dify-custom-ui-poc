# ArtifactUI ページ分割・印刷プレビューバグ報告書

本報告書は、AI（LLM）から返却されたHTMLアーティファクトにおける「プレビュー画面で紙が長くなりすぎる」「印刷プレビューで見切れる」という2つの現象についての原因と解決策をまとめたものです。

---

## 1. プレビュー画面で「紙が縦に長くなりすぎる」バグ

### 症状
プレビュー画面（ArtifactPanel）において、A4サイズの用紙の本来の比率を無視して、中身のテキスト量に合わせて用紙の下部が無限に拡張されてしまう現象。

### 原因
LLMが出力した以下のCSS指定と、LLM自体の「文字量の計算ができない」という特性のミスマッチが原因です。
```css
/* LLMが生成・適用したCSS */
.page {
  /* ... */
  min-height: 297mm;
  /* ... */
}
```
LLMは文字量がA4に収まるかどうかの絶対的なピクセル計算ができないため、すべての文章を1つの `<div class="page">` の中に無理やり出力していました。
その結果、実際の文字量がA4（297mm）を超過した場合、CSSの `min-height`（最小の高さ指定であって最大を制限していない）の仕様により、中の文字に合わせて用紙の枠ごと下方向へ自動的に拡張されてしまいました。

---

## 2. 印刷プレビューで「下部が見切れてしまう」バグ

### 症状
印刷を行おうとした際、プレビュー画面では伸びていたコンテンツ領域が強制的にA4の1枚の高さでカット（見切れ）されてしまい、2ページ目以降が印刷されない現象。

### 原因
こちらも同様にLLMが生成したCSS（特に印刷専用スタイルである `@media print` の指定）が直接の引き金となっています。
```css
/* LLMが出力した問題のCSS */
html, body { 
  overflow: hidden; /* はみ出た領域を強制的に隠す指定 */
}

@media print {
  .page { 
    height: 296.8mm; /* 印刷時は高さを「ピッタリ1枚分」に強制固定 */
  }
}
```
印刷時にはプレビュー画面の `min-height` と異なり、`height: 296.8mm` という**絶対値で高さが固定**されました。
さらに全体に `overflow: hidden` がかかっていたため、はみ出して次のページへ流し込まれるはずだった文章がすべて非表示（カット）されてしまい、「1枚目で表示が途切れる」という結果を引き起こしていました。

---

## 3. 真の根幹原因：「html-artifact-guide.md」の設計上の問題

上記のバグを引き起こすCSS（`overflow: hidden;` や `@media print` での高さいじり、自分でページを計算して分割する指示など）は、すべて **`public/html-artifact-guide.md` （LLMに与えられた生成ガイドライン）の中で、明確にLLMに出力するよう指示されていた内容** でした。

LLMは指示を守れなかったのではなく、「ガイドラインに書かれたルールを極めて忠実に遵守・再現しすぎた結果、ブラウザの本来の改ページ機能を破壊してしまった」というのが正確な原因です。

### 全体的な解決策・推奨される運用
「LLM側で事前にページの高さを計算して `<div class="page">` を自力で分割する」というアーキテクチャは、文字サイズのブレなどを考慮すると破綻しやすいため推奨されません。

ガイドライン（`html-artifact-guide.md`）から以下の内容を撤廃・修正し、改ページや余白の計算はブラウザ（とフロントエンドUI）の標準機能に任せるよう設計を見直す必要があります。

1. **`html, body { overflow: hidden; }` の出力を禁止・削除する**
2. **`@media print` 内での `.page { height: 296.8mm; }` などの固定高指定を禁止・削除する**
3. LLMに自力での `page-break-after: always;` の挿入を強制しない（またはフロントエンド側で動的に文字量を計算して分割するロジックに改修する）
