# バグレポート: HTMLスライドのレイアウト崩壊（連結・余白・重なり）

## 1. 概要
HTMLスライド型アーティファクトの表示において、スライドが1枚に連結されてしまう、スライド内に不要なグレーの余白が生じる、要素が重なり合うという複数のレイアウト不具合が発生した。

## 2. 発生事象
1.  **スライドの連結**: 1枚目と2枚目のスライドが分離されず、1つの長いページとしてレンダリングされる。
2.  **内容の重なり**: 2カラム構成などのスライドにおいて、下部の要素が上部の要素に重なって表示される。
3.  **内側のグレー余白**: スライドの内側（特に下部）に、本来存在しない薄灰色の余白（背景色）が露出する。

## 3. 原因分析
### LLM生成物に起因する詳細な原因
*   **属性の欠落**:
    *   `splitArtifactPages.js` が期待する `page-break-after: always` 属性を **最初のスライド** に付与しなかったため、分割ロジックが機能せず1枚目と2枚目が連結された。
*   **CSSの衝突と設計不備**:
    *   `html, body` に対して設定された `background: #f0f2f5` (薄灰色) が、iframeの背景として絶対的な優先順位を持ってしまった。
    *   `.slide` コンテナが `margin-bottom: 20px` や `box-shadow` を持っていたため、iframeの `scrollHeight` が実際のコンテンツ（540px）を超えて計算され、下の余白部分から `body` の背景色が露出した。
*   **Flexboxの誤用**:
    *   スライド内部の `two-col` やその子要素（`highlight-card`）に対して、親要素の高さが未確定な状態で `height: 100%` を多用した。結果として、コンテンツが垂直方向に溢れた際、後続の要素（黄色いテキスト `p`）が絶対配置のように手前の要素の上に重なる現象を引き起こした。

### 今後の対応策（恒久対策）
1.  **システムプロンプトの調整 (Prompt Engineering)**:
    *   スライド生成時、**すべての `.slide` Div** に対して `class="slide"` と `style="page-break-after: always;"` を例外なく付与するよう具体的に指示する。
    *   `html, body` への背景色指定を禁止し、`background: transparent` または指定なしを徹底させる。
2.  **CSSガイドラインの制定**:
    *   アーティファクト内部での `height: 100%` の使用を控え、`flex: 1` または `min-height` を推奨するよう指示を強化する。
3.  **フロントエンド側のガード（自動補正層）**:
    *   今回実装した **CSS注入（Style Injection）** をアーティファクトの仕様として定着させる。LLMが不適切なスタイルを生成した場合でも、表示時にフロントエンド側で強制的にリセット（透明化、マージン除去、重なり防止）を行うことで、表示の安定性を確保する。
    *   `splitArtifactPages` のように「クラス名ベース」の分割を行うことで、LLM側の属性忘れに対する耐性（レジリエンス）を持たせる。

## 5. 修正後の状態
*   全てのスライドが1枚ずつ正確に分離され、左右・下部の不要な背景色の露出が解消された。
*   カラムレイアウトが保持され、テキストが重なる現象が修正された。
