# **Json Document アーキテクチャ設計および実装案（v4.0 動的ページネーション＋SVG対応）**

## **1\. 概要と目的**

LLMに文書の構造化データ（JSON）を生成させ、フロントエンド（React）側で「A4用紙の印刷レイアウト」を動的に計算・描画するアーキテクチャです。

**最大の特徴（JS動的ページネーションと表現力）**:

* **フラットなJSON構造**: LLMは「ページ分割」を一切意識せず、フラットなブロック要素のみを出力します。  
* **動的レイアウトエンジン**: フロントエンド側で各要素の描画高さを計算し、A4サイズの仮想用紙（DOM）へ自動的に分割・配置（改ページ）を行います。  
* **無限の表現力**: リッチテキスト（Markdown \+ 許可されたHTML）に加え、AIによる**SVGの直接出力**をサポートし、自由度の高いインフォグラフィックや図解を実現します。

**期待される効果**:

1. **LLMの負荷とエラー率の最小化**: ページ制約や複雑なネストを意識させないため、ハルシネーションを防ぎます。  
2. **正確なページ番号付き目次の生成**: JSがレイアウトを計算した後に目次を生成するため、実際のページ番号と目次が完全に一致します。  
3. **完璧な印刷品質**: 画面上のA4プレビューと、実際の印刷・PDF化時の見た目が100%一致します。

## **3\. コア機能の仕様とルール**

### **3.1 ハイブリッド・リッチテキスト (rich\_text)**

テキストブロックには、Markdownと制限付きインラインHTMLのハイブリッドマークアップを適用します。

* style="..." による直接の色指定は禁止し、class='text-primary' のような事前定義済みのテーマユーティリティクラスのみを許可します。

### **3.2 SVG図解の直接出力 (svg)**

AIの空間把握・コーディング能力を活かし、SVGを直接出力させます。

* **高さ計算のための制約**: 改ページ計算を正確に行うため、スキーマで width と height（またはアスペクト比）のプロパティを必須とします。  
* **セキュリティ（XSS対策）**: 出力されたSVG文字列をそのままDOMに挿入するのは非常に危険なため、レンダリング前に DOMPurify 等のライブラリを通し、\<script\> タグや外部通信を完全にサニタイズ（無害化）します。

### **3.3 フロントエンドによる動的改ページ（ページネーション）アルゴリズム**

A4用紙の寸法（例: 210mm × 297mm をピクセル換算した値）を基準に、JSでブロックをページに振り分けます。

**処理フロー**:

1. **仮想描画 (Measuring)**: 画面外（visibility: hidden または専用の測定用コンテナ）にすべてのブロックを一度レンダリングし、各ブロックの実際のピクセル高さを測定します。  
2. **ページ分割 (Paginating)**:  
   * 測定した高さを順に加算し、A4用紙の許容高さ（上下の余白を除外）を超えた時点で改ページ処理を行い、次のページの配列にブロックを押し出します。  
   * ※ 見出しブロックの直後での改ページ（孤立行）を防ぐための「Keep-with-next（次と分離しない）」ロジックも実装します。  
3. **実描画 (Rendering)**: 分割されたページごとの配列データをもとに、画面上に複数のA4用紙コンポーネントを描画します。

### **3.4 自動目次（TOC）とページ番号の付与**

ページ分割ロジック（3.3）が完了した段階で、どのブロックが何ページ目に配置されたかが確定します。

* このタイミングで、type: "heading" のブロックとそのページ番号を抽出し、動的に「目次ブロック」を構築します。  
* 目次ページ自体を表紙の次に挿入し、必要に応じてページ分割の再計算を行います。

## **4\. 開発・実装ステップ（フロントエンド）**

この複雑なレイアウトエンジンを構築するための、段階的な実装手順です。

### **Step 1: パーサーと基本コンポーネントの実装**

* JSONデータを受け取り、type に応じて適切なReactコンポーネント（\<HeadingBlock\>, \<RichTextBlock\>, \<TableBlock\> など）を出し分ける BlockRenderer を作成します。  
* RichTextBlock では react-markdown と rehype-raw を用いてハイブリッドレンダリングを実装します。  
* この段階では改ページは行わず、単一の縦に長いコンテナにすべて描画して正しく表示されるか確認します。

### **Step 2: SVG対応とセキュリティの実装**

* DOMPurify をプロジェクトに導入し、サニタイズ設定（SVGタグの許可、scriptの排除）を定義します。  
* \<SvgBlock\> コンポーネントを作成し、JSON内の content (SVG文字列) をサニタイズした上で、指定された width / height のボックス内に安全に dangerouslySetInnerHTML で描画します。

### **Step 3: 高さ測定 (Measuring) ロジックの実装**

* ResizeObserver またはReactの useLayoutEffect を用いて、各ブロックがDOMにマウントされた直後に自身の高さ（offsetHeight ＋ margin）を親コンポーネントに報告する仕組みを作ります。  
* 隠しコンテナ（Off-screen container）を用意し、実際に画面に見せる前にすべてのブロックの高さを取得できるアーキテクチャを構築します。

### **Step 4: ページネーション・エンジンの構築**

* 取得した各ブロックの高さ配列をもとに、ページを分割するカスタムフック（例: usePagination）を実装します。  
* 計算結果に基づき、\<A4Paper\> コンポーネント（固定の縦横比と余白を持つコンテナ）を必要な枚数分レンダリングし、ブロックを流し込みます。

### **Step 5: 目次生成と最終調整**

* ページ分割完了後、配置情報から目次（TOC）データを生成し、2ページ目に挿入します。  
* 目次が挿入されたことによる高さの変化を吸収するため、必要であればエンジンを再計算（2パスレンダリング）させるロジックを組み込み、完成となります。
